# GitHub Templates

This repository exists as a neutral space for the Stan community to collaboratively decide on GitHub templates for Stan repositories.

The `.github` folder contains the current drafts. Everything else is intentionally empty.

## Scope

- Which issue and PR templates to adopt for Stan repositories
- What default content and structure each template should contain
- Any additional files or conventions worth discussing (e.g., Contributing.md, AI_POLICY.md)

## Current status

- Initial templates are available in `.github`
- GitHub templates (`.md`) are preferred over GitHub forms (`.yml`) — agreed in the [Stan Slack `#developer` channel](https://mc-stan.slack.com/archives/C7V03NJHL/p1773220124970069)
- Discussion is ongoing — open an issue, PR etc. to contribute

## Previewing the templates

1. Three example issues have been created corresponding to the three templates (Bug, Feature, General)
2. Create a new Issue: Go to **Issues** → **New Issue**, then you'll see each template listed as a separate option

Alternatively, browse the files directly in the [`.github`](.github) folder.
